# 更新日志

本文件记录豆包工作室的重要功能变化。

## [2.2.0] - 2026-07-14

### 视频任务可靠性

- 视频模型、时长和画面比例改为精确校验；配置未成功写入豆包页面时，任务不会继续提交。
- 新增视频能力预检：本地已知的额度耗尽、登录失效、人工验证、账号暂停和冷却状态会在执行前阻止任务。
- `15s` 视频配置显示会员权益风险提示，但不会绕过或伪造平台会员、额度、风控与服务端校验。
- 扩展会员、额度、真人脸、内容审核和生成失败的识别；明确限制出现后立即停止长时间轮询，且不扣减预计额度。
- 修复阻断检测将用户提示词误判为平台失败的问题：提示词成功提交后刷新检测基线，保留视频响应缓存。

### 调度与产物

- 修复全局暂停后残留运行锁导致的恢复失败；调度器异常路径会释放队列锁并显示恢复失败原因。
- 浏览器工具栏新增当前对话视频提取下载入口，手动生成的视频可解析、下载并更新对应账号的 Seedance 预计额度。
- 人工补提取超时视频时会补记额度；已完成任务重复提取不会重复扣减。

### 平台基础

- 新增共享 `@doubao-studio/contracts` 纯类型包，统一主进程、渲染进程与 Electron API 的领域模型和 IPC DTO。
- 新增 Capability API Schema、CLI/MCP 映射设计和视频能力预检的单元测试覆盖。
- 工程验证扩展至 401 项测试，并保持 IPC 契约、contracts 边界、类型检查与生产构建统一校验。

## [2.1.0] - 2026-07-12

### 运行稳定性

- 修复 IPC 注册边界，避免热重载或重复初始化导致通道重复注册。
- 统一账号、任务与自动化重试的调度入口，补齐账号引用、进程生命周期和 Webview 定时器清理。
- 强化任务异常后的资源释放与恢复路径，减少重复执行和遗留运行状态。

### 工程质量

- 工程检查改用 TypeScript AST，交叉校验主进程 IPC 与 preload 调用契约，并检查重复注册、方向错误、动态通道和顶层调用风险。
- 引入 ESLint 质量基线，`validate` 统一执行类型检查、静态检查、IPC 契约检查、测试和构建。
- 引入 Vitest，并抽取版本比较、CSV、错误分类、依赖判断、账号调度评分等纯逻辑测试接缝。
- 新增 92 条核心回归测试，覆盖 CSV、调度、依赖、错误分类和 JSON 备份恢复。

### 平台化准备

- 新增共享 contracts 类型迁移设计，明确主进程、渲染进程与未来 Agent API 的类型边界。
- 共享类型设计采用私有、纯类型 workspace contracts 包，保持 Electron 构建入口与运行时模块解析不变。
- 新增 G-302 能力协议设计任务，后续将为 CLI、MCP 与其他 Agent 提供稳定、安全的公共 Schema。

## [2.0.0] - 2026-07-11

### 项目化工作空间

- 新增项目数据模型、项目 IPC 和 Zustand 项目状态。
- 新增顶部项目切换器和新建项目入口。
- 旧任务启动时自动迁移到默认项目。
- 新建任务和 CSV 批次自动归属当前项目。
- 任务列表、批次管理和产物中心按项目隔离。
- 新增项目概览、项目完成率和脱敏项目包导出。

### 账号资源池

- 新增账号调度启停、权重、偏好模式和手动冷却字段。
- 账号菜单支持暂停调度、恢复调度和冷却 30 分钟。
- 智能评分加入账号权重、偏好模式和手动冷却。
- 排队任务所在账号不可用时，自动迁移到评分最高的健康账号。

### 日志与诊断

- 新增最多保留 5000 条的持久化运行日志。
- 自动记录任务开始、完成、暂停和结构化失败。
- 新增日志中心，支持级别与关键词筛选。
- 数据完整性检查覆盖 JSON 解析和项目引用。

### 备份与项目交付

- 新增完整备份导出和恢复，覆盖项目、账号配置、任务、下载、诊断、日志和设置。
- 备份明确排除 Cookie 和网页登录 Session。
- 新增项目包导出，隐藏远程产物地址并保留任务结构。
- 数据 Schema 升级至版本 6。

### 发布与更新

- 新增应用内 GitHub Release 更新检查。
- 新增 GitHub Actions 类型检查和生产构建工作流。
- 新增标签触发的 Windows 安装包构建与 Release 上传流程。
- 清理不存在的打包资源配置，提升 CI 打包成功率。

### 界面

- 产品标识升级为 v2.0。
- 顶部更多菜单增加项目概览、运行日志、数据检查、完整备份、恢复和更新检查。

## [1.5.0] - 2026-07-11

### v1.1.1 稳定性修复

- 新增主进程持久化任务锁，同一任务或账号不能被多个执行器重复占用。
- 清理程序异常退出后遗留的活动锁和运行状态。
- 应用界面重新载入时将活动任务安全转为暂停，避免重复提交。
- 修复全局暂停、继续和队列恢复之间的竞态。
- 运行历史最多保留 20 次结果、错误类型与实际耗时。

### v1.2 独立执行协调器

- 新增 `AutomationEngine`，统一管理任务预留、账号互斥、取消控制器和锁释放。
- 调度器获得主进程任务锁后才允许创建运行快照和启动 Webview 自动化。
- 取消、完成、失败和暂停统一释放执行器资源。
- 任务详情显示运行锁、运行历史、任务批次和前置依赖。

### v1.3 产物中心

- 新增统一产物中心，汇总历史图片、视频和文件产物。
- 支持提示词、任务 ID、类型和有效性筛选。
- 新增远程地址轻量验证，识别有效、过期、不可用和验证超时。
- 关联下载记录，显示产物是否已保存到本地。
- 支持预览、单项下载、重新验证和打开原豆包对话。

### v1.4 CSV 批次与基础工作流

- 新增 CSV 文件导入，支持中英文表头和带引号字段。
- 支持模式、模型、时长、比例、参考图片、参考音频和账号名称。
- 新增批次 ID、数据来源、前置任务和依赖策略字段。
- 支持 `all_done` 与 `all_finished` 两种依赖策略。
- 调度器只在依赖条件满足后启动下游任务。
- 新增任务批次页面和失败任务批量重试。
- 仓库新增可直接编辑的 CSV 示例模板。

### v1.5 页面适配系统

- 新增带版本号的豆包页面适配规则包。
- 新增页面兼容性自检，覆盖输入、提交、视频、模型、时长、比例、上传、验证和产物能力。
- 自检结果按账号脱敏保存，最多保留最近 50 份。
- 支持导入结构受限的 JSON 适配规则包。
- 规则更新自动保存最近 5 个历史版本，并支持一键回退。
- 规则包仅允许选择器数组，不允许执行外部脚本。

### 文档与发布

- README 更新到 v1.5，增加产物中心、CSV 工作流、任务锁和页面适配说明。
- 更新项目结构、使用步骤、CSV 模板链接和后续路线图。

## [1.1.0] - 2026-07-11

### 任务运行与恢复

- 新增持久化任务运行快照，记录运行 ID、执行次数、输入配置、当前阶段、阶段开始时间和最近心跳。
- 新增准备账号、创建对话、切换模式、配置参数、上传素材、填写提示词、提交、验证、生成和产物提取等细分阶段。
- 程序重启时自动识别上次未完成的任务，并安全迁移为可恢复的暂停状态。
- 修复“全部暂停”后调度器仍可能自动启动下一条任务的问题。
- “继续所有任务”现在会将暂停任务重新排队并恢复调度。
- 所有长等待均接入 AbortSignal，可立即暂停，不必等待视频超时。
- 失败任务保存结构化错误类型及是否可恢复信息。

### 视频自动化

- 每次启动和重新运行任务前强制创建新对话。
- 优化视频模型、时长和画面比例的选择与确认流程。
- 15 秒请求补丁增加独立启停状态，兼容自动任务和手动网页生成。
- 视频产物最长等待提升至一小时，并降低轮询与日志频率。
- 增加额度耗尽、会员限制、真人脸限制、审核失败和生成失败检测。
- 触发机器人验证后进入独立等待阶段；验证完成后重新创建对话、上传素材并提交。
- 生成失败或长时间等待时支持手动暂停和稍后提取。

### 产物管理

- 新增产物元数据，记录任务、运行 ID、账号、对话地址、发现来源和发现时间。
- 重新运行任务时保留历史产物记录。
- 手动提取旧任务前自动导航到对应豆包对话，降低错误产物匹配概率。
- 视频地址提取增加网络缓存、页面媒体、视频 ID 和播放信息等多通道候选。
- 产物预览支持图片放大和视频播放。

### 下载系统

- 新增持久化下载记录，保存下载状态、文件路径、文件大小和失败原因。
- 新增下载记录弹窗和单项重试。
- 下载请求使用任务所属账号的独立 Session，保留必要的登录上下文。
- 程序退出时尚未完成的下载会在下次启动后标记为可重试失败。
- 改进视频和图片扩展名判断、空文件检测及下载超时提示。

### 账号调度

- 新增账号登录状态、验证状态、连续失败、累计成功/失败和冷却时间。
- 连续失败账号自动进入冷却，验证中和登录失效账号暂停调度。
- Seedance 额度耗尽账号自动标记并沉底。
- 自动指派综合任务负载、额度余额、连续失败、冷却和手动置顶进行评分。
- 账号列表显示额度预测、累计成功/失败和实时运行阶段。

### 批量生产

- 所有非运行状态任务均可编辑提示词、参考图片、音频和视频配置后重新运行。
- 新增完整任务模板保存和复用。
- 新增 `{{变量}}` 批量展开，每行变量可创建一组独立任务。
- 已完成任务支持重新运行，同时保留历史产物元数据。

### 统计、诊断与数据安全

- 新增运行统计，包括成功率、平均耗时、人工处理任务、下载失败和可用账号。
- 新增脱敏诊断包导出，隐藏提示词正文、完整素材路径和产物地址。
- 本地 JSON 改为临时文件写入，并在覆盖前生成 `.bak` 备份。
- JSON 主文件损坏时自动尝试读取备份。
- 修复保存下载目录会覆盖任务模板的问题。

### 开发体验

- 新增统一开发启动器，自动检测空闲端口并将实际地址传递给 Electron。
- 修复 Vite 固定使用 `5173` 时因旧进程残留导致整个开发环境启动失败的问题。
- Electron 主进程开发地址支持 `VITE_DEV_SERVER_URL`。

- 建立 Electron、React、TypeScript、Vite 和 Zustand 桌面应用基础架构。
- 实现多账号 Session 隔离、账号列表和内嵌豆包网页。

- 支持对话、图片、视频和音乐模式。
